**Business workflow and task definition**

The workflow I chose is summarizing meetings into action items.

The user is a project manager or team member who needs to quickly review meeting outcomes and track next steps.

The system receives meeting notes, a transcript, or a rough summary of a discussion as input.

The system should produce a short meeting summary, a list of action items, the person responsible for each task if mentioned, and any deadlines or follow-up points.

This task is valuable to automate because meetings often generate a large amount of written information that takes time to review manually. Automating this process can save time, improve clarity, and help teams avoid missing important tasks or deadlines.
